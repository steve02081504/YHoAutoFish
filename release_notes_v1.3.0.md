# YHoAutoFish v1.3.0

本版本重点加入来源验证、授权状态入口、服务器群列表和机器人菜单能力，并继续保留原有自动钓鱼、记录、图鉴、自动更新等功能。

## 主要更新

- 新增来源验证：用户需要在官方 QQ 群通过 `/bind 绑定码` 完成来源绑定后才能使用核心功能。
- 新增授权状态按钮：标题栏可直接查看绑定状态、绑定码、复验状态和官方群入口。
- 新增后台静默复验：程序运行中固定 1 分钟复验一次；服务器临时不可达时，离线宽限固定 5 分钟。
- 新增服务器群列表：QQ 群入口和来源验证弹窗从授权服务器读取公开群列表与邀请链接。
- 新增机器人菜单：群内可使用 `/yho help`、`/yho group`、`/yho rules`、`/yho status` 等命令。
- 新增管理员后台能力：授权查询、设备释放、授权吊销、群入口配置、关键词治理、违规提醒文案和审计日志。
- 优化授权稳定性：在线复验成功会顺延授权 token 有效期，避免正常用户周期性重新绑定。
- 优化旧绑定码处理：本地已有授权 token 时不再轮询旧绑定码，减少“绑定码已经领取过授权”的误提示。
- 修复发布依赖：PyInstaller 明文包显式包含授权缓存所需的 Windows DPAPI 依赖。

## 升级说明

- 直接覆盖旧版本目录即可，原有 `config.json`、钓鱼记录、截图和日志不会被发布包主动覆盖。
- 首次使用 v1.3.0 时需要完成来源验证；换设备时需要联系管理员释放旧设备后重新绑定。
- 请只从 GitHub/Gitee 官方 Release 下载，不要从闲鱼、拼多多、发卡网等第三方渠道获取程序。

## 发布文件

- `YHoAutoFish-v1.3.0-windows.zip`：完整 Windows 发布包。
- `YHoAutoFish-v1.3.0-windows.zip.001`、`.002` 等：Gitee 国内源分卷包，每卷不超过 100MB。
- `latest.json`：自动更新清单，请和发布包一起上传。

## 运维提醒

- 服务端需要同步更新 `/opt/yho-auth/server/auth_service.py`，否则客户端授权复验顺延和 `/yho group` 公开群汇总回复不会生效。
- `ADMIN_API_TOKEN`、`ONEBOT_ACCESS_TOKEN`、`QQ_HASH_SALT`、服务端 `.env`、SQLite 数据库和私钥不得上传到 Release 或 Git。
